Food For Thought

Details:

Food for Thought is a multi-user blog that I made to practice writing server-side code in Python. The app features secured user signup and login functions made from scratch using various Python modules for hash and cookies. Users can post thoughts, comment and upvote other people's thoughts, and edit or delete their own posts. Unlogged users are restricted from viewing, commenting or upvoting posts.

Features:
  • Runs on Google Cloud App Engine
  • Utilizes a GQL database with data hosted by G Cloud Datastore
  • Hash and salt used for password security using sha256 algorithm
  • HMAC used for cookie values hashing
  • Webapp2 for route handling
  • Jinja2 for rendering templates
  • Semantic UI for simple, intuitive styling

Note: This web app is best viewed on desktop. Viewing on mobile devices can cause errors.

YelpCamp

Details:

YelpCamp is a multi-user web app that I made to explore the power of Node.js, Express, MongoDB and npm packages. This app features secured user signup and login functions and allows logged-in users to post, edit and delete campgrounds. Users can also post, edit and delete comments for each campground. Users can give addresses to campgrounds and have their locations shown on a Google Map. This app also features beautiful images which slow it down significantly since I'm using free database and server hosts.

Features:
  • Hosted on Heroku servers
  • NoSQL database (MongoDB)
  • Mongoose for configuring MongoDB models
  • Database hosted on mLab
  • Node.js used as environment for writing server-side code
  • Express for handling routes
  • Method-Override for handling update and delete functions
  • EJS for templating
  • Passport.js for password hash and salt
  • Express-Sessions for configuring cookies
  • Geocoder with Google Maps API for rendering locations of campsites
  • Flash messages to notify errors when User is redirected from a page
  • Bootstrap to provide responsive design that is great on mobile

Q Blogger

Details:

Q Blogger is a simple and universal blog that I worked on to practice using MongoDB and implementing basic CRUD functions on my data. This app allows a user to create blog posts, retrieve posts with content in it's last-modified state, update posts with new content and delete post objects from the database.

Features:
  • Hosted on Heroku servers
  • NoSQL database (MongoDB)
  • MongoDB data models made with Mongoose
  • Database hosted on mLab
  • Node.js for server-side code
  • Express for handling routes
  • Method-Override for handling update and delete functions
  • EJS templates
  • SemanticUI for basic page styling

Note: For best experience, view this web app on a desktop device.

Patatap Clone (Musical Keyboard with Animations)

Details:

This is a clone of a popular web app called Patatap. Users click on a keyboard key and the app responds with a sound and colorful animation sprites. Musically-talented users can create mediocre melodies and live out their DJ dreams or even impress an uninterested crowd with this web app!

Features:
  • Hosted as a static page on Heroku servers with simple PHP
  • HTML5/CSS used for Paper.js canvas
  • Paper.js used to animate circles with unique features for each key
  • Howler.js implemented to trigger a specific audio file for each key

Note: This web app only functions on devices with a physical keyboard.

My To-Do List

Details:

My To-Do List is a simple to-do list I made to practice jQuery. Users can create to-do items, check off items and delete items from the list--all without the use of a database. The power of jQuery was put to the test, allowing me to use minimal code to manipulate HTML elements. The app does lack data persistence and lists will clear upon page refresh.

Features:
  • Hosted as a static page on Heroku servers with simple PHP
  • HTML5/CSS
  • jQuery implemented to provide logic, control HTML elements and alter css styles

Note: For best experience, view this web app on a desktop device.
